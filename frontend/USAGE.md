# 사용 방법 가이드

## 웹 앱 vs 안드로이드 앱

이 프로젝트는 **하이브리드 앱** 방식으로 구성되어 있습니다. 같은 코드베이스로 **웹 앱**과 **안드로이드 앱** 둘 다 사용할 수 있습니다.

## 웹 앱 사용 (기존 방식 그대로)

### 개발 모드
```bash
cd d:\Cursor\naver-login-app\frontend
npm run dev
```
- 브라우저에서 `http://localhost:5173` 접속
- 기존과 **완전히 동일**하게 작동합니다
- 모든 기능이 그대로 사용 가능합니다

### 프로덕션 빌드
```bash
npm run build
npm run preview
```
- 빌드된 웹 앱을 서버에 배포
- 기존 웹 배포 방식과 동일합니다

## 안드로이드 앱 사용 (새로 추가된 기능)

### APK 빌드
```bash
# 방법 1: 배치 파일 사용
build-android.bat

# 방법 2: 수동 실행
npm run android:build
```

### 안드로이드 앱 실행
1. Android Studio에서 APK 빌드
2. 생성된 APK를 안드로이드 기기에 설치
3. 앱 아이콘을 눌러 실행

## 중요한 점

✅ **기존 웹 앱은 그대로 유지됩니다**
- `npm run dev` → 웹 브라우저에서 실행 (기존과 동일)
- `npm run build` → 웹 배포용 빌드 (기존과 동일)
- 모든 기존 기능이 그대로 작동합니다

✅ **안드로이드 앱은 추가 기능입니다**
- 같은 코드를 안드로이드 앱으로도 사용 가능
- 웹 앱과 안드로이드 앱이 **독립적**으로 작동
- 하나를 수정해도 다른 하나에 영향 없음

✅ **Capacitor는 래퍼 역할만 합니다**
- 기존 웹 앱 코드를 그대로 사용
- 안드로이드 네이티브 기능만 추가
- 웹 앱 동작에는 전혀 영향 없음

## 개발 워크플로우

### 웹 개발만 하는 경우
```bash
npm run dev  # 기존과 동일
```

### 안드로이드 앱도 개발하는 경우
```bash
# 웹 개발
npm run dev

# 웹 빌드 후 안드로이드 동기화
npm run android:sync

# Android Studio 열기
npm run android:build
```

## 요약

| 항목 | 웹 앱 | 안드로이드 앱 |
|------|-------|--------------|
| 실행 방법 | 브라우저 | APK 설치 |
| 개발 명령어 | `npm run dev` | `npm run android:build` |
| 빌드 명령어 | `npm run build` | Android Studio에서 빌드 |
| 기존 코드 변경 | 없음 | 없음 |
| 독립성 | ✅ 독립적 | ✅ 독립적 |

**결론: 기존 웹 앱은 그대로 사용하고, 안드로이드 앱은 필요할 때만 빌드하면 됩니다!**
